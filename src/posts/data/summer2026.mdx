## Settings & UI Overhaul

* Added outline overheal health bars
* Updated the in-game chat font for better readability on high-resolution displays.

## Settings Menu Update

* Added keybindings menu to the modern Settings menu
  * Improved naming of a few keybinds for Team Fortress 2 and modern expectations. For example, "Duck" is now called "Duck / Crouch".
  * Added category filtering and search to keybindings
  * Command associated with the bind is now shown for discovery and search purposes
  * Enhanced user flow for rebinding keys
  * Remove irrelevant / unused keybinding options
* Added "Condensed View" to Graphics Quality menu. Now you can easily preview graphics settings changes while in-game through a minimal translucent menu.
* Added crosshair preview. Supports previewing crosshair type, scale, color and gap
* Added Outlines section in General to customize outlines settings
* Added "custom wav" option in hitsounds/killsounds to reduce confusion
* Added zoom sensitivity ratio to mouse settings
* Finding settings through search now highlights and scrolls to the setting found for better navigation
* Added descriptions/tooltips to all settings
* Fixed a few settings bugs:
  * HTML MOTD setting removed as it is no longer available
  * Fixed Remember Previous Weapon between lives not working
  * Fixed maximum value on Medic Auto caller option being too high
  * Fixed unknown settings values being blank
  * Fixed select options box not showing properly if it was at the bottom with no room
  * Fixed misalignment on checkbox settings
* Added icons for the various sections
* Added option to disable featured class, falling back to the classic class images
* Added confirmation prompt when binding to a key that is already used by another action
* Added voice receive volume to Communication > Voice instead of it just being in volume settings
* Added option to enable chat notification sound
* Labeled Beepo as default hitsound and killsound

## New Communication Wheel

* Added radial menu for voice menu (Z, X, C by default) as a beta quality feature
  * Can use number keys as before to select an option, or hold the button and drag the mouse to select an option
  * Can use tf_radial_voicemenu_concise 1 to split up the voice menu options into groups of 4 (rather than needing to reach all the way for 9)
  * The radial menu is fully customizable, allowing for various communication bubbles, message text templates and the classic voice commands.
  * Customization and additional options are not implemented at this time, and we are only using the classic voice commands
  * Radial menu options can be conditional, for example PASS Time specific options or Medic specific options

## Casual Matchmaking Update

* Reduced the match timer from 35 to 25 minutes
* Fixed not being able to kick players on the MOTD screen with match assigned teams
* Adjusted team shuffle algorithm:
  * Player score discrepancy will now adjust based on Team Series Score, in order to better detect situations where a team runs away with the lead in terms of game state/objectives
  * Bot difficulty will now adjust a skill curve to further compensate for team balance in situations where shuffling human players is not sufficient. This prioritizes making the losing team's bots stronger, and then falls back to making the dominant team's bots weaker.
* Random crits now reset back to on after each match (as was done initially). This better ensures we don't linger past user preferences when a new match starts.
* Changed the number of bots that fill in for players for Casual play to 75% of a full lobby (a 9v9 for 24 player servers)
* Fixed flag announcements playing during match summary (e.g., flag returned)
* Fixed the team that won the series not being used for MVP at the end of a match
* Fixed some MVP stats being set to 0 rather than the correct value
* Fixed first player not spawning in the correct spawn in matchmaking games
* Fixed players not being respawned when a new series falls back to waiting for players

## Outlines Revamp

* Reworked outline rendering to fix various issues and improve performance
  * Loosely based upon work from PazerOP (thank you!)
* Fixed outlines disappearing when a player was ubered or had another forced material override
* Fixed outline color not correctly inputting into HDR / tonemapping
* Added support for outlines for only parts of an object visible, or only behind a surface
* Changed the display rules of some outlines based upon this new system:
  * Spawn/spectator outlines no longer draw an outline when visible
  * Payload carts no longer draw an outline when visible
  * Dropped weapons now reliably draw only when visible
  * Flags no longer draw an outline when visible
  * Sticky Bombs (except for the Scottish Resistance) no longer draw an outline when visible
  * MvM Tanks no longer draw an outline when visible
  * MvM Money no longer draws an outline when visible

## HDR / Bloom Update

* Reworked how HDR lighting and colors are computed. Visibility should be improved, unnatural bright and dark spots eliminated, and colors should have improved range and contrast.
* Reworked how bloom is computed. The fidelity and relevance of the bloom should be improved, and performance impact of bloom decreased.
* Improved performance of post processing
